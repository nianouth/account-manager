## 📖 简介

账号管理器是一个强大的 Chrome 浏览器扩展，帮助开发者、测试人员和多账号用户高效管理不同环境（开发、测试、生产）下的多个账号。通过智能网站匹配、一键自动登录、账号模板复用等功能，让账号管理变得便捷高效。

### 为什么选择账号管理器？

- **⚡ 高效便捷**：智能识别登录页面，一键自动填充并提交表单
- **🎯 精准匹配**：基于 URL 的智能网站匹配，支持通配符和 SPA 路由
- **🎨 双界面设计**：弹出窗口 + 悬浮面板，适应不同使用场景
- **📋 账号模板**：创建可复用的账号模板，关联多个环境
- **💾 多格式导入导出**：支持 JSON/CSV 导出、模板下载、AI 辅助生成

## ✨ 功能特性

### 核心功能

#### 1️⃣ 网站管理
- **多网站多环境**：管理开发、测试、生产等多个环境
- **智能 URL 匹配**：
  - 精确匹配：`https://example.com/login`
  - 通配符匹配：`https://example.com/login/*`
  - URL 规范化：自动忽略查询参数和锚点
- **自动填充环境信息**：添加环境时自动读取当前标签页的标题和 URL
- **自定义登录按钮**：高级设置中配置登录按钮的 ID 和 Class

#### 2️⃣ 账号管理
- **完整的账号信息**：用户名、账号、密码、备注
- **⭐ 账号收藏**：收藏常用账号，自动置顶显示
- **📋 一键复制**：快速复制账号和密码到剪贴板
- **🔍 实时搜索**：带防抖的关键词快速筛选
- **重复检测**：添加/编辑时自动检测同网站下重复账号

#### 3️⃣ 账号模板
- **创建可复用模板**：将通用账号组织为模板组
- **环境关联**：模板可关联到多个环境，自动出现在对应网站
- **模板标记**：悬浮面板中模板账号显示"模板"标记
- **独立管理**：Popup 中独立的模板 Tab 页，支持完整 CRUD

#### 4️⃣ 一键自动登录
- **智能表单识别**：自动定位用户名和密码输入框
- **多选择器支持**：兼容邮箱、手机号、用户名等多种字段
- **React/Vue 兼容**：使用原生 value setter 兼容受控组件
- **SPA 支持**：支持无 `<form>` 标签的单页应用登录

#### 5️⃣ 双界面模式

**弹出窗口（Popup）**
- 四 Tab 导航：账号、网站、模板、设置
- 完整的管理界面，适合添加、编辑、配置等操作
- Guardian 蓝色主题，现代化 UI 设计

**悬浮面板（Floating Panel）**
- 仅在匹配的登录页面显示
- 快速访问当前网站账号和关联模板账号
- 支持拖拽调整位置，位置自动保存
- 支持直接编辑和删除账号
- 智能定位：自动检测登录表单位置，放置在不遮挡的一侧

#### 6️⃣ 数据导入导出
- **JSON 导出**：完整数据备份，包含版本信息和时间戳
- **CSV 导出**：Excel 自动识别 UTF-8 编码
- **JSON 导入**：支持合并导入，数据不会丢失
- **下载导入模板**：标准 JSON 模板，按格式填写后导入
- **AI 提示词下载**：下载提示词文件，通过 AI 生成符合格式的导入数据

#### 7️⃣ 快捷键
- `Ctrl/Cmd + K`：聚焦搜索框
- `Ctrl/Cmd + N`：添加新账号
- `Esc`：关闭模态框
- `Alt + A`：切换悬浮面板展开/折叠
- `Tab/Enter/Space`：键盘导航账号列表

## 🚀 快速开始

### 系统要求
- Chrome 浏览器 88.0 及以上版本
- 支持 Windows、macOS、Linux

### 安装步骤

1. **下载项目**
   ```bash
   git clone https://github.com/nianouth/account-manager.git
   cd account-manager
   ```

2. **打开 Chrome 扩展程序页面**
   - 访问 `chrome://extensions/`
   - 或点击菜单：更多工具 → 扩展程序

3. **启用开发者模式**
   - 在扩展程序页面右上角，开启"开发者模式"

4. **加载扩展**
   - 点击"加载已解压的扩展程序"
   - 选择项目文件夹 `account-manager`
   - 扩展将自动加载并显示在扩展列表中

## 📖 使用指南

### 首次使用

#### 步骤 1：添加网站
1. 点击扩展图标，打开弹出窗口
2. 切换到"网站"Tab，点击添加按钮
3. 填写网站信息（名称和 URL 会自动从当前标签页填充）：
   - **网站名称**：如"开发环境"、"生产环境"
   - **登录页面 URL**：完整的登录页面地址
   - **高级设置**：（可选）配置登录按钮 ID/Class
4. 点击"保存"

#### 步骤 2：添加账号
1. 切换到"账号"Tab，选择对应网站
2. 点击"添加账号"按钮
3. 填写账号信息：用户名、账号、密码、备注（可选）
4. 点击"保存"

#### 步骤 3：一键登录
1. 访问对应的登录页面
2. 悬浮面板会自动显示（智能定位，不遮挡表单）
3. 点击账号的"登录"按钮
4. 系统自动填充并提交登录表单

### 日常操作

- **🔍 搜索账号**：在搜索框输入关键词，实时筛选
- **✏️ 编辑信息**：在 Popup 或悬浮面板中编辑
- **⭐ 收藏账号**：点击星标，常用账号自动置顶
- **📋 复制信息**：一键复制账号或密码到剪贴板
- **💾 备份数据**：设置 Tab → 导出（JSON/CSV）
- **📥 恢复数据**：设置 Tab → 导入 JSON 文件

### 使用账号模板

1. 切换到"模板"Tab，创建模板组
2. 在模板组中添加通用账号
3. 编辑环境，在"关联模板"中选择需要的模板
4. 关联后，模板账号自动出现在该环境的账号列表和悬浮面板中

## 🛠️ 技术架构

### 技术栈
- **核心**：原生 JavaScript (ES6+)
- **框架**：Chrome Extension Manifest V3
- **存储**：Chrome Storage API (local)
- **UI**：Shadow DOM 隔离、CSS 设计令牌、CSS 动画
- **浏览器 API**：Tabs API、Scripting API、Runtime API

### 项目结构
```
account-manager/
├── icons/                    # 扩展图标 (16/48/128/512px)
├── images/                   # 截图示例
├── utils/                    # 工具模块
│   ├── index.js              # 模块导出入口
│   ├── dom-utils.js          # 安全 DOM 操作
│   ├── toast.js              # Toast 通知系统
│   ├── dialog.js             # 模态对话框
│   ├── validation.js         # 表单验证
│   └── url-matcher.js        # URL 匹配工具
├── manifest.json             # 扩展配置 (Manifest V3)
├── background.js             # Service Worker
├── content.js                # 内容脚本（悬浮面板注入）
├── content-utils.js          # 内容脚本共享工具
├── popup.html                # 弹出窗口 HTML
├── popup.js                  # 弹出窗口逻辑
├── popup.css                 # 弹出窗口样式
├── panel.css                 # 悬浮面板样式
├── design-tokens.css         # 设计令牌（统一变量）
├── config.example.json       # 配置示例
├── CHANGELOG.md              # 更新日志
└── README.md                 # 项目文档
```

### 架构设计

#### Service Worker (background.js)
- 扩展生命周期管理（安装、更新）
- 监听标签页更新，自动匹配网站
- 处理 URL 变化和网站切换

#### Content Script (content.js + content-utils.js)
- 注入悬浮面板到匹配的登录页面（Shadow DOM 隔离）
- 实现表单自动填充和提交（兼容 React/Vue）
- 监听 URL 变化（包括 SPA hashchange）
- 面板拖拽、位置保存、智能定位

#### Popup (popup.html/popup.js/popup.css)
- 四 Tab 管理界面（账号、网站、模板、设置）
- 网站和账号的 CRUD 操作
- 数据导入导出（JSON/CSV）
- 实时搜索和筛选

#### 设计系统 (design-tokens.css)
- 统一的设计令牌（颜色、间距、字体等）
- Guardian 蓝色主题（#3B7DD8）
- Popup 和悬浮面板共用一套设计体系

### 安全机制

- **XSS 防护**：使用 `textContent` 和 `createElement`，避免 `innerHTML`
- **Shadow DOM 隔离**：悬浮面板样式与页面完全隔离
- **权限最小化**：仅请求 `storage`、`activeTab`、`scripting` 权限
- **CSP 策略**：配置内容安全策略防止代码注入

## ❓ 常见问题

### Q: 悬浮面板不显示怎么办？
**A:** 请检查：
- 当前页面 URL 是否与网站配置的登录页面 URL 匹配
- 确保已添加网站并正确配置登录页面 URL
- 刷新页面重试

### Q: 自动登录失败怎么办？
**A:** 请检查：
- 使用浏览器开发者工具（F12）检查登录按钮的实际 ID 和 Class
- 在网站高级设置中填写对应的选择器
- 某些网站可能有额外的验证机制（如验证码）

### Q: 支持哪些网站？
**A:** 理论上支持所有使用标准 HTML 表单的网站，包括 React/Vue 等框架构建的 SPA 应用。如果遇到不兼容情况，请提交 Issue。

### Q: 数据存储在哪里？
**A:** 数据存储在 Chrome 本地存储中，不会上传到任何服务器。建议定期使用导出功能备份数据。

### Q: 如何批量导入账号？
**A:** 在设置页中：
1. 点击"下载模板"获取标准 JSON 格式模板
2. 或点击"AI 提示词"下载提示词，通过 AI 生成导入数据
3. 按格式填写后，使用"导入"功能导入
