本 skill 指导创建独特的、生产级别的前端界面，避免泛化的"AI 烂大街"审美。用真实可运行的代码实现，在审美细节和创意选择上精益求精。

用户会提供前端需求：一个组件、页面、应用或界面。他们可能会说明用途、受众或技术限制。

## Chrome 扩展技术约束

本项目是 Chrome 扩展（Manifest V3），有以下硬性限制，**必须遵守**：

- **纯原生技术栈**：只能使用 HTML/CSS/JS，不可引入 React、Vue 等框架，不可使用 npm/构建工具。
- **CSP 限制**：`script-src 'self'`，禁止内联脚本、`eval()`、外部 CDN 脚本。所有 JS 必须在独立 `.js` 文件中。
- **字体限制**：不能通过 `@import` 或 `<link>` 加载外部字体（如 Google Fonts）。如需自定义字体，必须将字体文件本地打包到扩展中并用 `@font-face` 引用。否则使用系统字体栈中有质感的选择（如 Georgia、Palatino、Garamond 等衬线体，或 SF Pro、Helvetica Neue 等无衬线体），通过精心的 `font-weight`、`letter-spacing`、`font-feature-settings` 调教出个性。
- **Popup 窗口尺寸**：Chrome 扩展 Popup 最大约 800×600px，设计必须在此范围内工作。
- **Shadow DOM**：悬浮面板（Content Script 注入的 UI）运行在 Shadow DOM 内，样式完全隔离。设计时需确保样式自包含，不依赖宿主页面的任何样式。
- **动画只能用纯 CSS**：不可引入外部动画库。善用 `@keyframes`、`transition`、`animation-delay` 编排动效。
- **已有设计系统**：项目已有 `design-tokens.css` 定义了 CSS 变量体系，新设计应优先复用和扩展现有变量，保持一致性。

## 设计思维

**写代码之前**，先理解上下文，并锁定一个大胆的美学方向：

- **用途**：这个界面解决什么问题？谁在使用？
- **风格基调**：选一个极端方向——极致简约、极繁主义、复古未来感、有机自然、奢华精致、玩具感、杂志编辑风、粗野主义、装饰艺术/几何、柔和粉彩、工业实用主义……风格无穷无尽，以这些为灵感，设计出真正契合方向的作品。
- **记忆点**：什么让这个界面令人**难以忘怀**？用户最终会记住什么？

**关键原则**：选定清晰的概念方向，精准执行。大胆的极繁和克制的极简都能成功——关键在于**有意为之**，而非强度本身。

然后实现可运行的代码（HTML/CSS/JS），要求：
- 生产级别，功能完整
- 视觉震撼，令人难忘
- 风格统一，有明确的审美立场
- 每一个细节都精雕细琢

## 前端审美准则

重点关注以下几点：

- **字体排版**：在系统字体的限制内发挥创意。通过精心的字重搭配、字间距、行高、`text-transform`、`font-variant` 等属性，让普通字体也能焕发个性。标题与正文形成层次对比。
- **色彩与主题**：坚守一套统一的审美。优先使用和扩展 `design-tokens.css` 中的 CSS 变量。主色调 + 锐利强调色的组合，远胜于平均分配的多色方案。
- **动效**：用纯 CSS 为交互效果和微交互添加动画。聚焦高价值时刻：一个精心编排的页面加载动画（配合 `animation-delay` 错落出现），比到处散布的微交互更令人愉悦。善用悬停状态变化带来惊喜。
- **空间构图**：在 Popup 的有限空间或悬浮面板内，通过精巧的间距、层次和视觉节奏创造呼吸感。小空间更考验排版功力。
- **背景与视觉细节**：营造氛围和层次感，而不是默认纯色背景。运用纯 CSS 实现的渐变、几何图案（`repeating-linear-gradient`）、多层 `box-shadow`、`backdrop-filter`、精致边框等手法。

**绝对禁止**使用以下泛化 AI 审美：俗套配色（尤其是白底紫色渐变）、千篇一律的布局和组件模式、缺乏上下文个性的模板化设计。

创意地理解需求，做出真正为这个场景量身设计的意外之选。每个设计都应该是独一无二的。

**重要提示**：让实现的复杂度匹配审美愿景。极繁主义设计需要大量动画和效果的精密代码；极简或精致的设计需要克制、精准，以及对间距、排版和微妙细节的细心关注。优雅来自于对愿景的完美执行。
